# OctoAcme Project Management Docs

Welcome! This README provides a single entry point to all project management process documentation and practices adopted by OctoAcme.

## Project Management Process Overview

OctoAcme's project management approach is structured around practical, iterative delivery with clear ownership and transparency. Our process emphasizes customer value, data-driven decisions, and continuous improvement.

### Core Principles
- **Customer-first**: Prioritize customer value and usability in all decisions
- **Iterative delivery**: Break work into small, testable increments for rapid feedback and course correction
- **Clear ownership**: Each project has a named Project Manager and Product Manager; each team member has defined responsibilities
- **Data-informed decisions**: Measure impact, track metrics, and iterate based on evidence
- **Psychological safety**: Encourage feedback, learning, and blameless retrospectives
- **Transparent communication**: Regular syncs, demos, and status updates keep all stakeholders informed

### Project Lifecycle
OctoAcme projects follow a structured five-phase lifecycle:

1. **Initiation**: Validate business need, align stakeholders, and define success criteria using a lightweight One-pager
2. **Planning**: Break work into shippable increments with clear acceptance criteria, estimates, and a prioritized backlog
3. **Execution**: Build, test, and iterate using daily standups, weekly syncs, and a project board for visibility
4. **Release**: Deploy to production with pre-release checks, smoke tests, and clear rollback procedures
5. **Close & Retrospective**: Capture learnings and convert them into actionable improvements

### Key Workflows
- **Team Rhythm**: Daily standups (15 min), weekly delivery syncs, and demo/reviews at sprint or milestone end
- **Pull Requests**: Small PRs (≤400 lines preferred) with issue links, acceptance criteria, and at least one approval
- **Quality Gates**: Unit tests, integration tests, security scanning in CI, end-to-end smoke tests, and manual QA
- **Risk Management**: Identify, assess, mitigate, and monitor risks via a Risk Register with regular escalation reviews
- **Communication**: Weekly PM + Product Lead syncs, twice-weekly standups, monthly stakeholder updates, and ad-hoc escalations

### Core Roles
- **Project Manager**: Coordinates delivery, schedules, risks, and communications
- **Product Manager**: Defines outcomes, prioritizes the backlog, and measures success
- **Developers**: Implement features, collaborate on design and testability, assist in planning
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs, approvals, and business context

## Getting Started

**New to OctoAcme?** Start here:
1. Read this README to understand our principles and lifecycle
2. Review [Project Management Overview](octoacme-project-management-overview.md) for a quick orientation
3. Depending on your role, read the relevant process docs (see links above)
4. Ask your Project Manager or Product Lead for clarification on any process

- Initiation → Planning → Execution & Tracking → Release → Retrospective

Refer to the relevant process document at each stage for checklists, templates, and guidance.
